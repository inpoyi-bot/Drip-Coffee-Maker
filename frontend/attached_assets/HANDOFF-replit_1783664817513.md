# Handoff · Replit Agent(阶段 1:垂直切片起骨架 + 本地跑通)

> 贴给 Replit Agent 的任务书。配套契约文件 `FRONTEND.md` 必须一并提供,它是唯一真值。
> 你(Replit)负责**架子和管道**;IP 敏感的精密零件由下游 Codex 精装,你**留占位、别实现**。

## 0. 项目背景(一句话)

一个手冲咖啡诊断教练的前端。用户逐杯报告冲煮反馈,后端 agent 做收敛诊断(爬山),前端渲染诊断结果和收敛轨迹。后端已存在(ADK + Gemini + SQLite),**你不碰后端**,只做前端并对接它的本地 API。

## 2. 后端对接(本地 ADK,只读不改)

- 后端是本地 ADK,启动方式(**你不需要启动它,用户会启,你只对接**):
  ```
  ./.venv/bin/adk api_server agents --port 8000 --session_service_uri="sqlite+aiosqlite:///./sessions.db"
  ```
- 前端连 `http://localhost:8000`。
- **repo 里已有一个本地 demo frontend 连这个后端**——请参考它的 API 调用方式和 CORS 处理,别从零猜 ADK 的接口协议。
- ⚠️ **CORS**:前端(:8080)连后端(:8000)跨端口,会触发 CORS。确认 ADK api_server 允许前端 origin;参考现有 frontend 怎么解决的,别忽略这条。

### ★ 硬约束:API 地址必须可配置,不准硬编码
后端地址(`http://localhost:8000`)**必须**做成环境变量 / 配置项(如 `VITE_API_BASE_URL`),**绝不**散落硬编码在各组件里。
> 理由:将来若要分享 demo(用隧道给本地后端一个临时公网 URL),只需改一个配置就能切地址。硬编码会让未来分享要挖遍所有组件。这是低成本保住可分享性的关键约束。

## 3. ★ 部署红线:目标是"本地跑通",不是"Replit 云端在线"

**明确:本阶段"部署通路"= 本地 `npm run dev` 能跑起来 + 能连 localhost:8000 + `npm run build` 不报错。不是上 Replit 云端在线。**

原因:后端只在用户本地 localhost 跑,Replit 云端的前端够不到用户机器的 localhost(网络拓扑决定,不是配置问题)。

- ❌ **不要**尝试把前端部署到 Replit 云端并让它在线连后端——会连不通,别在这上面耗时间或编造假后端糊弄。
- ❌ **不要**为了"让它在线"而 mock 一个假后端替代真 ADK。垂直切片必须连真后端证明管道通。
- ✅ 交付标准就是本地能跑通真管道。

## 4. 垂直切片:冷启动 → 杯2 欠萃诊断

走 demo 干净弧线的前两杯,证明整条管道:

**杯1 · 冷启动**
- 用户在冷启动屏填:烘焙度=浅焙、烘焙日期=8天前、粉量=15g、磨豆机=锥刀电动磨、起始研磨=像粗砂糖那么粗。
- 前端**把这些组装成自然语言整句**发给 agent(见 §6),agent 回起点 seed。

**杯2 · 欠萃诊断**
- 用户在反馈录入屏报:很酸、酸完发空收尾短、寡淡水感、流速快约1分40秒、床面浅、没挂粉环。
- 前端组装成自然语言发给 agent,agent 回诊断(欠萃→磨细)+ 一条 record_cup。
- 诊断结果屏渲染回来的 record_cup(此阶段用占位文案,显示字段值即可)。
- Cup Timeline 出现两杯;「我的」轨迹页折线图能画出这两个点(占位灰色)。

## 5. ★ 轨迹图红线:占位色,颜色逻辑别写死

「我的」轨迹页你负责**图表能动**(接 recharts、喂真数据、画出折线、响应式、mobile 竖排)。但:

- **全部用占位色(如统一灰 `--slate`)**,**绝不**自主给数据点上色、加渐变、做花哨强调。
- **颜色逻辑别写死在图表里**——留成可替换的样式钩子。下游 Codex 会按 `FRONTEND.md` §5「收敛时刻」精装:收敛点用 `--converge` 点亮、且全页仅此一处彩色、震荡段中性。
- 你若把颜色写死,Codex 精装时要拆你的硬编码。留活口。

> 为什么这么切:轨迹页是这个产品的核心展示物,收敛色是**语义化的唯一色**(标记"用户到达"的成就时刻),不是装饰。你负责渲染力,语义强调留给按契约精装。

---

## 6. 关键接口约束:前端发给 agent 的是自然语言,不是结构化 payload

用户在表单点完结构化选项后,前端**把选择组装成自然语言整句**再发给 agent(如"这杯还是有点酸,比上杯变好,床面平,杯壁没挂粉环"),**不发结构化 JSON payload**。

> 理由(重要,别改):agent 的 eval 输入是自然语言,agent 在自然语言上被测试调优。发结构化 payload 会让生产输入 ≠ eval 输入,eval 失去对生产的保护。这是契约红线,见 `FRONTEND.md` 屏2「提交形态」。

---

## 7. 应用 design tokens 时避开的坑

- 骨架**微冷中性**(白/灰带极轻冷调),**不要**暖奶油/陶土色——那是"AI 咖啡 app"默认三件套,刻意避开。
- 数据(杯序、时间、gradient)用**等宽字体**,这是仪器气质的关键。
- **分层密度**:触点屏(冷启动/反馈录入)走足留白;IP 屏(诊断/轨迹)克制配色但高信息密度。见 `FRONTEND.md` §4。
- 收敛色 `--converge`(青绿)本阶段**别用**——它是下游精装的 signature,你全程用中性色。
