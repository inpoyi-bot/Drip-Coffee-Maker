# FRONTEND.md · Design Tokens v2 · 深色科技方向

> **状态**:v2 视觉方向 · 待 Chillin review
> **方向升级(已决)**:v1 浅色微冷仪器骨架 → **v2 深炭底 + 冷色光晕 + 发光数据**的科技仪器感。
> **为什么升级**:v1 "够用但朴素";v2 目标是**作品集/面试拿得出手的水平**(Chillin 设计背景)。方向锚定三张参考图:深炭底 + 冷光晕(视觉锤)+ 数据为主角发光高亮。
> **⚠️ 留痕**:这是方向升级不是推翻。v1 的这些约束**全部保留**:teal 语义唯一、等宽数据字体=仪器感、反差对靠文案不靠色、分层密度、signature 只标收敛。v2 只翻转底色系 + 新增光晕 + 数据发光。旧版 tokens 存档不删(决策日志习惯)。

## 1. 色板 v2(深色)

### 骨架(深炭五阶,微冷)
| token | hex(建议·待审) | 角色 | 用在哪 |
|---|---|---|---|
| `--void` | `#0E1013` | 最深背景 / 页面底 | 全站主背景(近黑,微冷,非纯黑) |
| `--surface` | `#16181D` | 卡片 / 面板底 | 诊断卡、轨迹卡、表单区块(比底稍亮一阶) |
| `--surface-raised` | `#1E2128` | 抬升面 / hover 态 | 选中卡、悬浮、需要跳出的面 |
| `--line` | `#2A2E37` | 分隔线 / 边框 | hairline 分隔、卡片边 |
| `--text-dim` | `#7A828E` | 次级文本 / 数据标签 | confidence 标签、刻度、辅助信息 |
| `--text` | `#E4E8ED` | 主文本 | 正文、标题(近白微冷,非纯白,减少刺眼) |

### 强调 + signature(色相由 Chillin 定,明度为深底提亮)
| token | hex | 角色 | 用在哪 |
|---|---|---|---|
| `--converge` | `#2ECC8F` | **★ 收敛色(唯一语义色)** | **仅**收敛/满意时刻:满意终止、轨迹收敛点、graduation |
| `--converge-deep` | `#1B6F4C` | 收敛色暗版(Chillin 原值) | 收敛区域填充、渐变低端、非焦点的收敛关联元素 |
| `--interact` | `#3D7BD9` | 交互色(选中/focus/可点) | 表单选中态、focus 描边、可交互强调 |
| `--interact-deep` | `#0A4CA2` | 交互色暗版(Chillin 原值) | 选中态背景填充、渐变低端、hover 底 |

> **色值决策留痕**:Chillin 定色相 teal=`#1B6F4C`(墨绿)、blue=`#0A4CA2`(宝蓝)。这两个在深炭底上明度不足、无法"发光跳出"(深色强调+深色底=对比不足,违背"收敛点是焦点/选中态明显"的目标)。**采路 A:保留色相,为深底提亮。**
> - `--converge` `#2ECC8F`:Chillin 墨绿的**提亮版**(同绿色相,明度拉到深底能发光)。焦点/亮点用它。
> - `--converge-deep` `#1B6F4C`:**保留 Chillin 原值**,用于收敛区域的**渐变低端/填充**(暗版做底、亮版做焦点,一个色相两个明度=深度)。
> - blue 同理:`#3D7BD9` 亮版做焦点描边,`#0A4CA2` 原值做填充底。
> **一个色相两明度 = 既尊重你的色相选择,又满足深底发光需求,还自带渐变深度**(参照图3:亮线 + 暗填充是同色相不同明度)。
> ⚠️ 提亮值 `#2ECC8F`/`#3D7BD9` 待你看**实际渲染**微调——纸上 hex 判断不准,渲染出来你觉得不够亮/太亮再调。

### 光晕(视觉锤 · 新增)
| token | 值(建议·待审) | 用法 |
|---|---|---|
| `--glow-cool` | 冷绿(呼应 converge)→冷蓝(呼应 interact)极暗弥散径向渐变 | 全站背景氛围层。用你的绿/蓝色相的极暗弥散版,和强调色同源,统一 |

> **光晕规范(红线)**:
> - **极暗、极弥散**:是背景"呼吸感",不是显眼色块。透明度极低,大范围羽化。
> - **数据/内容永远是清晰前景**:光晕绝不降低文字/图表可读性。前景对比度优先。
> - **全站统一位置逻辑**:形成视觉锤——建议统一从某一致方位弥散(如每屏右上或中心偏上),别每屏乱放。
> - **冷色**:和 teal/交互蓝同色系,不掺暖。

## 2. 字体(v1 保留,深色微调)

| 角色 | 字体 | 深色调整 |
|---|---|---|
| Display | 骨感无衬线(Inter Display / Söhne),Regular–Medium | 深底用 `--text` 近白;克制 |
| Body | Inter / Söhne,中文思源黑体/苹方 | 深底可读性:字重可略提(深底细字重会糊),但别过粗 |
| **Mono/Data** | 等宽(JetBrains Mono / IBM Plex Mono) | **仪器感承重墙,v1 保留**:杯序/时间/gradient/研磨读数用等宽,深底下等宽数据发光=仪器感 |

> ⚠️ 深色下字重:纯细字重在深底容易发糊,body 可比浅色版略提一档,保证可读。别为"精致"牺牲可读。

## 3. 质感(深色新增规格)

| 元素 | v2 做法 |
|---|---|
| 卡片 | `--surface` 底 + `--line` 细边;**可加极微妙的边缘发光/内发光**(参照图:卡片边缘一线冷光),但克制 |
| 选中态 | `--interact` 冷蓝描边(1.5-2px)+ 极淡冷蓝内发光 + radio 填充冷蓝。深底下描边发光=科技感 |
| 收敛强调 | `--converge` teal:实心点 + 发光光晕(参照图3 高亮点的 glow),全页唯一 |
| 圆角 | v1 保留:`--radius-sm:6px`/`--radius-md:10px`(精密但不冷酷) |
| 分隔 | `--line` hairline;深色下靠明度差分层,不靠阴影 |
| 输入框 | 深底 + `--line` 边,focus 时 `--interact` 冷蓝描边发光(参照图:输入框冷色 focus) |

| 屏 | 密度 | 目的 |

## 5. Signature 收敛时刻(v1 精神保留,深色强化)

| 触发 | v2 视觉 |
|---|---|
| satisfied/收敛 | `--converge` teal 首次出现,深底下发光更强 |
| 轨迹收敛点 | teal 实心发光点 + 光晕,全页唯一彩色焦点;前面爬升段中性 |
| graduation | 同色语义呼应 |

> 语义留痕(v1 保留):teal=成就标识非装饰,绝不用于导航/普通按钮。深色下它更亮,更要守住唯一性,否则满屏发光 teal 就廉价了。

---

## 6. 反差对红线(v1 完全保留)

诊断状态区分**绝不靠颜色**承担——flavor_mismatch⟂taste_unaddressable⟂axis_limit 等必须靠**文案**分开,颜色至多辅助。深色下尤其注意:别因为"深色好看"就用不同色块区分诊断分支,那会把判别力交给视觉。teal 仍是唯一语义色,只标收敛正向态。
