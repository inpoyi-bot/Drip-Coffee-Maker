> **背景**:深色 v2 tokens 已定稿(`FRONTEND-tokens-v2-dark.md`)。逐屏铺开,本次做**诊断页**——你 IP 的核心展示屏。
> **⚠️ 前置**:等前一屏(冷启动)完成后再做,保持小步。
> **红线**:纯前端视觉。不动后端/eval/record_cup/诊断逻辑。terminate_reason L1 文案、turn_type 文案、fallback、confidence 逻辑全部**不改内容**,只改视觉。分支 `frontend-replit-v1`。
> **特殊性**:这屏比表单复杂——它有"结构化字段(仪表盘)+ 教练说明(人话)+ 反差对文案(IP)"多个层次。深色化要处理好主次,尤其守住反差对纪律。

## 1. 应用深色 tokens + 光晕(同前屏,视觉锤一致)

- 底 `--void`、卡 `--surface`/`--surface-raised`、线 `--line`、文本两阶
- 全站冷光晕同套(统一位置逻辑)

## 2. 两种卡的层次(核心:教练说明 vs 结构化字段)

诊断页有两类信息,深色下要主次分明又协调(呼应你定的层次分离:结构化为主干、自由文本为人话主沟通):

**教练说明卡(agent 人话,你定 b:显眼但整理好)**
- 深色卡(`--surface`),**显眼**(初学者主要靠这段人话指导)
- markdown 渲染好、行距舒适、可读性优先(深底下 body 字重按 tokens 规格,别糊)
- 是"教练在跟你说话"的感觉——温度、清晰

**结构化字段卡(仪表盘:状态/变化/动作/下一步/研磨)**
- 深色卡,字段标签等宽(`--text-dim`),值用 `--text`——**仪表盘/仪器读数感**
- 密度可比教练卡高(IP 屏=密)
- chip/标签深色化:深底 + hairline,别用彩色块

**层次关系**:两卡视觉上可区分(教练卡偏"对话温度",字段卡偏"仪器读数"),但同属一个诊断结果,协调不割裂。别再是之前那种"两个一样的灰气泡"。

## 3. ⚠️ 反差对纪律(深色下尤其守,红线)

- **诊断分支的区分绝不靠颜色块**:flavor_mismatch ⟂ taste_unaddressable ⟂ axis_limit_underextracted 等,靠**L1 文案**区分(你已定稿的七条),颜色至多中性辅助
- **深色下的诱惑**:深色好看,容易想给不同诊断态配不同色块(如换豆用橙、下一步用蓝)——**绝不**。那会把判别力交给颜色,毁掉你 evals 守的判别纪律
- terminate_reason 的七条文案**原样**(不改字),只是深色排版

## 4. confidence 星级(深色版,独立呈现)

- 星级(★★☆)深色下显示:实心星用 `--text` 亮、空心用 `--text-dim`。**别用彩色星**(不抢焦点、不混语义)
- **独立呈现**:星级是校准维度,和诊断结论分开(你 FRONTEND 定的"认知边界≠校准边界"),别把星混进诊断句
- low 附"需要更多信息判断"——深色下这句用 `--text-dim`,附注不喧宾

## 5. 收敛态的 teal(诊断收敛时)

- 当诊断是 **satisfied / 收敛**(terminate_reason=satisfied 等收敛正向):`--converge` `#2ECC8F` 可点缀强调该诊断卡(它是收敛时刻)
- **其他诊断态**(欠萃/调整中/probe/换豆/plateau...):**中性**,不用绿
- 守收敛色唯一:只有"到位/收敛"才配绿,"还在调/有问题"都中性

## 6. rationale 折叠(为什么这样判断)

- 保持折叠(非主出口,你 v1 定的)
- 深色下:折叠标题 `--text-dim`,展开后正文可读;它是补充,不与主诊断争视觉

## 7. 分层密度(IP 屏=密)

诊断页是 IP 屏,按 tokens 走**密**:结构化字段用等宽 + 紧凑对齐 + hairline 分隔,营造高级仪表盘感。但教练说明卡那部分保持可读留白(人话要舒适)。密的是仪表盘,不是人话。
