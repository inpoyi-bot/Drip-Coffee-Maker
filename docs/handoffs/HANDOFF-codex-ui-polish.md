# Handoff · Codex:UI/UX 打磨批次(试用前体验收口)

> **背景**:功能主链路已通(冷启动→反馈→probe→轨迹),真实使用后发现一批交互/视觉问题。这批是**试用前的体验收口**,纯前端。
> **⚠️ 全批红线**:纯前端。不动后端、不改 agent、不碰 eval、不动 record_cup 结构。
> **工作位置**:分支 `frontend-replit-v1`,前端 `frontend/artifacts/coffee-coach/`。
> **风格约束**:守 FRONTEND.md 的微冷中性 + Blue Bottle 克制方向。这批是"在既定方向内提精度",不是加花哨。收敛色 `--converge`(teal)语义唯一,仅收敛时刻用,别挪作他用。
> **节奏**:逐个小步,每步停下等确认。建议顺序见末尾。

---

## P1 · 选中态无反馈(可用性)

**问题**:表单选项(烘焙度、研磨粗细等)选中后视觉变化极弱(仅背景微灰、radio 圈几乎不变),用户看不出选没选上。

**改**:新增一个**次级强调色 token**(冷灰蓝,和微冷骨架同色温、比纯灰有一点色相),用于选中态。
- 选中项:次级色描边(1.5-2px)+ radio 圆点填充次级色 + 背景极浅次级色调
- 未选:保持中性
- **⚠️ 不用收敛色 teal**(那是收敛时刻专用,别稀释语义);次级色要和 teal 明显区分
- 建议色值(可微调,用户会 review 实际效果):冷灰蓝 `#5B7085` 或近似(饱和度低、偏冷)。给出你选的色值和它在 tokens 里的命名(如 `--select-accent`)。

---

## P2 · 漏选不提示哪项(可用性)

**问题**:必填项没选完时按钮置灰(对),但不提示**差哪项**,用户要自己找。

**改**:克制的必填提示(**不用红色报错**,守 anti-overclaim 风格):
- 未选的必填项旁,低调标记(如小字"必填"或次级色描边提示)
- 点灰按钮时:高亮/滚动到第一个未选的必填项
- 保持克制:不要满屏红字、不要弹窗警告

---

## P3+P5 · agent 自由文本重构(可读性,核心)

**问题**:
- P3:agent 输出的 markdown 没渲染,`**粗体**` 显示成字面 `**`,列表 `*` 成星号,整段挤成一坨(见冷启动屏、诊断屏)
- P5:前端把 agent 一轮的多个 text 片段各渲染成一个独立灰气泡,出现"两个同级气泡",且两段都以"好的"开头,重复机械

**改(按层次分离哲学)**:
1. **结构化为主**:诊断/下一步的高层级信息用 `recordCup` 结构化字段渲染(现有的状态/方向/动作/下一步/研磨/星级卡片)——这是可靠的 IP 来源,保持。
2. **agent 自由文本 → 合并成一张"教练说明"卡**:
   - 把一轮里 agent 的**所有 text 片段合并成一张卡**(不再多个气泡),不从自由文本猜语义
   - 这张卡**显眼、整理好**(用户是初学者,需要 agent 的人话指导——这是主要沟通,不是次要补充)
   - **渲染 markdown**:`**x**`→粗体、`*`→列表项、段落正确分行、行距舒适。彻底解决"成坨"
   - **去重复开场**:多段合并时,顺掉第二段起的重复开场词(如第二段以"好的"开头就去掉),让合并后读起来是一段连贯的话。⚠️ 简单规则即可,别做复杂关键词清洗(脆弱)。
3. 层次:教练说明卡(人话、可读)+ 结构化字段卡(仪表盘辅助),两者层次分明,不再是一堆混杂气泡。

> 注:agent 话术整体简洁度(偏啰嗦)是单独的后端优化项(动 prompt+重跑 eval),**不在本批**。本批只做前端展示重构。

## P4 · turn_type 状态文案(语义)

**问题**:状态栏「这轮调整」是 turn_type 直译,用户看不懂。

**改**:状态栏 turn_type 按下表显示用户文案(栏标签保留「状态」):

| turn_type | 用户文案 |
|---|---|
| `seed` | 起点配方 |
| `adjust` | 调整研磨 |
| `probe` | 追问确认 |
| `terminate` | 调整完成 |

> 跨屏同源:状态栏这些文案在诊断卡、Timeline、轨迹卡片都用同一份(和 diagnosisCopy.ts 同思路,可放同一处维护)。

## P6 · 轨迹图横轴缺 1(图表 bug)

**问题**:折线图横轴只有 2、3,第 1 杯没显示,和下方卡片列表对不上。

**改**:横轴从第 1 杯开始,所有杯都在图上(1、2、3…)。

---

## P7 · 轨迹图收敛点视觉用反(⚠️ signature 错位,重要)

**问题**:当前/收敛杯(最后一杯)显示成**空心白点**,teal 实心点却在中间的非收敛杯上——signature 完全用反,收敛点反而最弱。

**改(严格按 FRONTEND.md §5)**:
- **收敛/当前杯(最后一杯)**:实心 `--converge`(teal)大点,**全页最强视觉焦点**——这是"你到达了"的成就标识
- **前面的爬升/震荡点**:中性色小点,不抢戏
- **全图唯一的 teal 亮点必须在收敛处**,不在别的杯
- ⚠️ 这是护城河那屏的核心视觉,是这批里最重要的一个,别再错位

---

## P8 · 轨迹卡片排序:正序 + 顶部当前摘要(信息架构)

**问题**:卡片顺序 + 缺当前状态快速入口。

**改**:
- **主体正序**(第1杯→第2杯→第3杯):讲收敛过程叙事(这页是护城河展示,正序讲"怎么一步步爬山收敛的")
- **顶部单独一个"当前状态"摘要**:把当前杯(最后一杯)的关键信息(当前研磨、当前状态、是否收敛)置顶做一个摘要块,方便快速看"现在在哪"
- 兼顾:顶部看当前 + 下方正序看过程

> 顺带(可选):副标题「这些杯子来自你保存的记录,不是当前对话上下文」是护城河的关键陈述(证明持久化非聊天错觉,SPEC §5.3),现在是灰色小字。可略微加强其存在感(不必大改)。

## 建议顺序(小步)

1. **P3+P5**(教练说明卡 + markdown 渲染)—— 最影响可读性,先做,做完文字立刻清爽
2. **P7**(收敛点 signature 修正)—— 核心视觉,单独做单独验
3. **P6**(横轴)+ **P8**(排序+摘要)—— 轨迹图一起
4. **P1**(选中次级色)+ **P2**(必填提示)—— 表单交互
5. **P4**(turn_type 文案)—— 可并入任意步

每步停下给用户验。P7 尤其要单独验(signature 不能再错)。
