# 第一块 · 反馈页

反馈页本质是表单,**复用冷启动屏的深色表单规格**(选中态、层次、光晕),额外处理感官词按钮的分类。

## 1. 基础(同冷启动)
- 深色 tokens + 冷光晕(视觉锤一致)
- 表单选中态:`--interact` `#3D7BD9` 冷蓝描边发光 + radio/checkbox 填充冷蓝
- 层次:字段标签等宽、留白节奏、选项卡质感
- 触点屏=疏(初学者录反馈,别密)

## 2. 感官词按钮(A/B/C,视觉不改逻辑)
- 感官词按钮(酸/发苦/发涩/薄/纸板味/说不清)深色化:深卡 + hairline + 选中冷蓝描边
- **逻辑不变**:A类直接落账、B类(酸/薄)触发 probe、C类(说不清)开放 probe——**只改视觉,触发逻辑原样**
- **无甜按钮**保留(防用户拍板萃取毕业)——视觉上和其他一致,别特殊化
- 苦/涩输入层分开(视觉上是两个独立按钮,别合并)

## 3. 双补位(视觉区分不合并)
- 「没注意看」(数据缺失)vs「尝到了但说不清」(触发 probe)——两个补位**视觉上可区分、不合并**(它们语义不同)
- 深色下用中性差异区分,别用彩色

## 4. probe 选项按钮(如果本轮是 probe 回应)
- probe 回应的选项按钮(P2酸收尾/P3绝对刻度/P4偏好/P5不甜)深色化:同表单按钮风格,冷蓝选中
- **逻辑/文案不变**(那些判别选项文案是 IP,原样),只改视觉

## 5. 客观读数(冲煮时间/床面/挂粉环)
- 这些客观字段:等宽数据字体(仪器感),深色仪表盘风
- 是"读数录入",视觉上可比感官词区更有仪器感

# 第二块 · Timeline(Cup Timeline 贯穿组件)

Timeline 是贯穿各屏的杯序进度组件(不是独立大屏)。深色化要保证它在任何屏都协调 + 收敛标识对。

## 1. 基础深色
- 深色适配:节点、连线、当前杯标识深色化
- 在各屏(诊断/轨迹/反馈)都协调,和所在屏的深色底融合

## 2. 节点状态(⚠️ 守收敛色纪律)
- **普通杯节点**:中性色(`--text-dim` 类小节点)
- **当前杯**:中性提亮(`--text` 亮)或极淡 `--interact` 冷蓝——**弱强调,不用绿**(和轨迹页当前杯一致)
- **收敛杯**:`--converge` `#2ECC8F` 发光节点——**唯一的绿**,只有真收敛才用
- 连线:中性;接近收敛处可渐变泛绿(呼应轨迹页折线做法)

## 3. 一致性(视觉锤)
- Timeline 的收敛标识做法**和轨迹页折线图一致**:收敛点绿发光、当前杯中性弱强调、普通杯中性
- 这样用户在 timeline 和轨迹图看到的"收敛"是同一套视觉语言,强化视觉锤

## 4. 等宽杯序
- 杯序号(1/2/3...)用等宽字体(仪器感),和轨迹图横轴刻度一致
